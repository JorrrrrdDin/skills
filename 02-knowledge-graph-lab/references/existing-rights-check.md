# Existing Rights Check

Use this when the user asks whether a public Knowledge Gravity Lab artifact may collide with existing patents, products, or private internal inventions.

This is not legal advice. Treat it as a product/design risk checklist.

## Lower-Risk Positioning

- Present the skill as note analysis, knowledge hygiene, and research workflow support.
- Use ordinary implementation ideas: text extraction, token overlap, link analysis, graph scoring, review queues, and report generation.
- Say "flags possible review items" rather than "detects attacks" or "guarantees clean memory."
- Say "topic gravity" as a metaphor, not as a protected control system.

## Higher-Risk Positioning

Avoid claims that sound like a protected production system:

- autonomous routing or cache-control behavior,
- guaranteed contamination defense,
- rollback-based operational safety,
- threshold-triggered memory-state rewriting,
- exact dynamic mass update formulas,
- exact post-split stabilization logic,
- claims of freedom-to-operate or non-infringement.

## Release Checklist

Before public release:

1. Remove unpublished claim language.
2. Remove exact private formulas and thresholds.
3. Avoid saying it is legally clear.
4. Avoid saying it implements any specific protected invention.
5. Keep examples focused on notes, papers, and ordinary text corpora.
6. Include "experimental heuristic tool" wording.
7. If commercial use is planned, do a formal patent/product review outside the skill.

## Useful Public Claim

"This skill helps organize messy research notes by surfacing central notes, noisy items, orphan notes, and overgrown topic groups."

## Avoided Claim

"This system provides legally protected, production-grade, contamination-resistant AI memory routing."
